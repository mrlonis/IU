  • What is the difference between n = 1 + 2 and n = "1" + "2"? If you were to display the contents of n, what would be shown on your page for each?
    • The difference between "n = 1 + 2" and "n = "1" + "2"" is that the first n will return 3 and the second n will return "12" due to data types.
  • What does the += mean/do? What did we use for in our JavaScript function?
    • The symbol '+=' is an assignment operator used to assign an addition for a variable. So, for example, if we have x = 2 and x += 3, x really equals 5. We used this symbol in our number counting JavaScript.
  • What two pieces of code/functions/attributes have we used the last few labs to indicate/identify where our JavaScript changes are going to happen?
    • We have used ID tages both in the attributes of a tag (for example "< p id="test" >") as well in the JavaScript code (document.getElementById("test").
  • What benefit comes from breaking down the construction of bigger projects into smaller pieces?
    • The benefit that comes from breaking down the construction of bigger problems into smaller pieces is that errors are less common and they're easier to find. It creates a more stable construction.
  • Why did we constantly check and test our code? What could go wrong if we did not test our code thoroughly?
    • We conastantly check and tested our code becuase if we didn't, one problem could've caused the entire JavaScript code to not work and it would have been a hassle to go back through and figure out where the bug is.
